Yonda API ドキュメント

読書記録にアクセスするための REST API です。

1ベース URL

本番 https://yonda.ktrips.net
ローカル http://localhost:8080

2認証

現在、認証は不要です（パブリックアクセス）。
Slack コマンド (/slack/command) は Slack 署名検証（SLACK_SIGNING_SECRET 環境変数）で保護されています。

3エンドポイント

書籍一覧

GET /api/v1/books 読書記録を条件でフィルタリングして返します。
パラメータ型デフォルト説明
statusstringallread 読了 / unread 未読 / in_progress 読中 / all
sourcestringallkindle / audible_jp / setagaya / all
qstring—タイトル・著者の部分一致検索
sortstringloan_date_descloan_date_desc / completed_date_desc / percent_desc / title_asc
limitinteger50最大 200
offsetinteger0ページオフセット

リクエスト例:

# 読了済みの Kindle 本（新しい順）
GET /api/v1/books?status=read&source=kindle&sort=completed_date_desc

# 読んでいる途中の本
GET /api/v1/books?status=in_progress

# タイトル検索
GET /api/v1/books?q=ハリーポッター&limit=10

  "success": true,
  "total": 42,
  "offset": 0,
  "limit": 50,

    "audible_jp": { "total": 40,  "read": 18, "in_progress": 2,  "unread": 20 },

書籍詳細

GET /api/v1/books/{catalog_number} ASIN や図書館番号で1冊を取得します。 
# Kindle の本を ASIN で取得
GET /api/v1/books/B0XXXXX
{
  "success": true,
  "book": { /* 書籍オブジェクト（上記と同じ形式）*/ }
}

// 見つからない場合
{ "success": false, "error": "該当する書籍が見つかりません" }  // 404

4Slack コマンド

POST /slack/command Slack Slash Command /yonda のエンドポイント。Slack 署名検証あり。

Slack App に Slash Command /yonda を設定し、Request URL にこのエンドポイントを指定してください。
環境変数 SLACK_SIGNING_SECRET の設定が必要です。

コマンド説明
/yonda read直近の読了済みの本（最大5冊）
/yonda reading読んでいる途中の本（進捗の高い順）
/yonda unread未読の本
/yonda stats統計情報（総冊数・ソース別など）
/yonda <キーワード>タイトル・著者で検索
/yonda helpヘルプを表示
