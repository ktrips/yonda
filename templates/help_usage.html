読書記録取込み

右上のハンバーガーメニュー（≡）から「読書記録取込み」を選択します。

  • メニューを開くと、図書館・Audible・Kindle の最終取得日時がソースごとに表示されます
  • 取得したいソースを選択して「取込み開始」をタップ
  • Cloud Run 環境では 毎日 06:00 / 12:00 / 18:00（JST）に自動取得されます

Yonda（読んだ？）— 読書記録一覧

タブ構成

  • Yonda: 自分の読んだ本・読中の本一覧
  • みんなのYonda: ユーザー全体の読書コミュニティ

フィルター・並べ替え

  • 検索: タイトル・著者のリアルタイム検索
  • ソース: 図書館 / Audible / Kindle / 紙の本
  • 状態: 読了 / 未読 / お気に入り / 評価（★）
  • 並べ替え: 取得日・読了日・著者・積読日数・評価など

表示モード

  • カード表示: 表紙・タイトル・概要をカード形式で表示
  • テーブル表示: 詳細情報を列形式で一覧表示。右端に非公開・非表示のチェックボックスあり

マイ・ランキング

Yonda タブ内に★評価順のランキングをインライン表示。年別フィルター付き。

本の詳細

本のカードまたはテーブル行をタップすると詳細モーダルが開きます。

  • ★評価: 個人評価を優先表示（Audible はカタログ評価にフォールバック）。タップするとレビューページへ
  • レビュー: 見出しまたはコメントを星の横に表示
  • Amazon・メルカリ: ボタンから外部サイトへ（アフィリエイトタグ付き）
  • 書評ポイント: AI生成または手入力のレビューポイント。📝 アイコンからレビューを書くページへ
    • 「手入力」「AI生成」ボタンで追加・編集
    • 各ポイントのコピーボタン付き
  • 紙の本: 「編集」「削除」ボタンで管理
  • 非公開・非表示: モーダル右下のチェックボックスで設定（→ 詳細）

みんなのYonda

他のユーザーの読書記録を閲覧できるコミュニティタブ。

  • 直近の読書メッセージを最大10件表示（1件あたり5冊まで）
  • ジャンル色: 本カードの背景・左ボーダー・丸バッジがジャンル色で表示
  • 表示順: ジャンルバッジ → ソースバッジ（図/A/K/P）→ 本の名前
  • 個人評価★・レビューがあれば著者の下に表示
  • 本をタップすると詳細モーダルが開く
  • 非公開に設定した本はここに表示されません

Yomu（何読む？）— 読みたい本検索

タイトル・著者名で検索するか、本の写真を撮って情報を自動抽出します。

検索

  • Amazon・Kindle・Audible・メルカリ・ブックオフ・図書館を横断リンク表示
  • 検索結果がない場合も外部リンクと「+紙の本」ボタンを表示

写真から検索

  • カメラで撮影または写真ライブラリから選択
  • AI（OpenAI/Gemini）またはバーコード（ISBN）・OCR で本の情報を自動抽出
  • 抽出した情報で検索・保存

+紙の本として保存

  • タイトルを入力またはヘッダー検索結果から「+紙の本」をタップ
  • Amazon リンクを貼ると著者・概要・表紙を自動取得
  • 写真を表紙として使用可能（自動圧縮）
  • 登録した本はYondaタブの一覧に表示（ソース: 紙の本）

Amazon 欲しいものリスト

  • アプリ設定 → Amazon設定 で Wishlist URL を登録すると本が表示されます
  • 「+Amazonリスト」「+紙の本」ボタンで追加した本がここに表示されます

AI推し — AI による本の提案

4つのモードから好みの方法で本の提案を受けられます。

簡単質問推し

性別・年代・職業・ジャンル傾向・読書頻度をスライダーで設定。AI が質問しながら本を提案します。

MBTI推し

自分の性格タイプ（MBTI）を伝えると、それに合った本を提案します。

強み診断推し

StrengthsFinder などの強み・得意なことを伝えると、それを活かす本を提案します。

読書履歴推し

自分の読了本・未読本リストを AI に渡し、最適な本を推薦理由付きで提案します。

  • API Key の設定が必要（アプリ設定 → AI設定）
  • 読了本と未読本の両方がある場合に機能します
  • 「もう一度おすすめを生成」で再提案可能
  • 推薦理由は本カードの右横に表示
チャット入力エリアは「読書履歴推し」モードでは非表示になります。

アプリ設定

右上のハンバーガーメニュー（≡）→「アプリ設定」から各種設定を変更できます。

Amazon設定

  • Wishlist URL: Amazon 欲しいものリストの URL を登録
  • アフィリエイトタグ: 設定すると全 Amazon リンクに自動追加（例: &tag=mytag-22）

AI設定

  • OpenAI API Key または Gemini API Key を登録
  • 書評ポイントの AI 生成・AI推しモードで使用

検索するアプリ

  • Amazon・Kindle・Audible・メルカリ・ブックオフ・図書館（所在地）のチェックで検索先を選択
  • 「＋追加」でカスタムアプリ（名前＋URL テンプレート）を登録可能
  • 設定はブラウザのローカルストレージに保存されます

非公開・非表示フラグ

全てのソース（図書館・Audible・Kindle・紙の本）の本に設定できます。デフォルトはブランク（公開・表示）。

フラグ効果設定場所
非公開 みんなのYondaに表示されない 本の詳細（右下）/ テーブル一覧（非公開列）
非表示 自分の一覧にも表示されない 本の詳細（右下）/ テーブル一覧（非表示列）
非公開にした本は、テーブルビューの「非公開」列でチェックを外すか、本の詳細から「非公開（みんなのYondaに非表示）」のチェックを外すと解除できます。

設定方法

  1. 本の詳細モーダルを開く（本をタップ）
  2. モーダル右下の「非公開（みんなのYondaに非表示）」をチェック
  3. 即座にサーバーに保存されます

テーブル表示でも各行の右端にある「非公開」列でチェックを変更できます。

外部 API（REST API v1）

認証不要で外部から読書データを取得できる読み取り専用 API です。Gmail アドレスでユーザーを指定します。

ベース URL: https://yonda.ktrips.net

プロフィール取得

GET /api/v1/users/{Gmail アドレス}/profile

ユーザーの名前・アバター・読了冊数を返します。

読書リスト取得

GET /api/v1/users/{Gmail アドレス}/books

パラメーターデフォルト説明
filtercompletedcompleted（読了）/ recent（直近）/ in_progress（読中）/ all
days7recent 時の日数（最大 365）
sourceallkindle / setagaya / audible_jp / paper / all
limit50最大件数（最大 500）
offset0ページネーション用オフセット

直近読了ショートカット

GET /api/v1/users/{Gmail アドレス}/recent?days=7&limit=20

全ユーザー統計（公開）

GET /api/public/user-stats

全ユーザーの名前・アバター・読了冊数を返します。ログイン不要。

本の各フィールド: title / author / genre / cover / source / completed / completed_date / rating / comment / isbn / asin / runtime_length_min

このアプリの作り方

Yonda は Cursor + Claude を使って週末だけで開発しました。同様のアプリを自分で作りたい方向けに、設計・実装・デプロイの全工程を解説した開発ガイドを公開しています。

  • マルチユーザー対応の仕組み（threading.local によるデータ分離）
  • 任意の市区町村図書館に対応できる汎用アダプタ設計
  • Audible・Kindle からの自動データ取得
  • Cloud Run へのデプロイ・定期同期の仕組み
  • Cursor + Claude を使った AI 駆動開発の進め方
