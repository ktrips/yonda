## 1. はじめに

### このガイドで作るもの

**yonda**は、読書記録を複数のソースから自動収集し、AIが書評・選書を支援するWebアプリです。

```
主な機能:
✅ Audible Japan の聴いた本を自動同期
✅ Kindle の読んだ本を自動同期
✅ 公共図書館（世田谷区立図書館）の貸出履歴を自動同期
✅ 紙の本をカメラ撮影 → AIでタイトル抽出して登録
✅ Google Books / Open Library で表紙・概要・ジャンルを自動補完
✅ AIで書評ポイントを自動生成
✅ AIとの対話で「あなたにぴったりな本」を選書
✅ Googleアカウントでログイン・複数ユーザー対応
✅ みんなの読書記録をコミュニティで共有
✅ Amazon アフィリエイトリンク自動生成
```

### スレッドローカルによるユーザーデータ分離

Cloud Runは複数リクエストを並列処理するため、どのユーザーのデータを読むかをスレッドごとに管理します。

```python
# library_service.py
import threading
_tls = threading.local()

def set_user_data_dir(path: Path) -> None:
    """リクエストごとのユーザーデータディレクトリをセット"""
    _tls.user_data_dir = path

def get_user_data_dir() -> Path:
    """現在スレッドのユーザーデータディレクトリを返す"""
    return getattr(_tls, 'user_data_dir', DATA_DIR)

def get_current_uid() -> Optional[str]:
    """data/users/{uid}/ のパスからUIDを抽出"""
    d = get_user_data_dir()
    try:
        rel = d.relative_to(DATA_DIR / "users")
        return str(rel).split("/")[0]
    except ValueError:
        return None
```

### mtime ベースのキャッシュ

同一リクエスト内では書籍データを繰り返しファイルから読まないようキャッシュします。

```python
_saved_caches: dict[str, Optional[dict]] = {}
_saved_cache_mtimes: dict[str, float] = {}

def load_saved() -> Optional[dict]:
    key = str(get_user_data_dir())
    max_mtime = _get_books_max_mtime()
    if _saved_caches.get(key) is not None and max_mtime <= _saved_cache_mtimes.get(key, 0.0):
        return _saved_caches[key]  # キャッシュ利用
    result = _load_saved_uncached()
    _saved_caches[key] = result
    _saved_cache_mtimes[key] = max_mtime
    return result

def invalidate_saved_cache() -> None:
    """保存後にキャッシュを強制無効化"""
    key = str(get_user_data_dir())
    _saved_caches.pop(key, None)
    _saved_cache_mtimes.pop(key, None)
```

## 5. Google OAuth認証の実装

### Google Cloud Consoleでの設定

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. 「APIとサービス」→「認証情報」→「OAuthクライアントIDを作成」
3. アプリケーションの種類: **ウェブアプリケーション**
4. 承認済みリダイレクトURI に追加:
   - `http://localhost:5002/auth/callback`（開発用）
   - `https://your-cloudrun-url/auth/callback`（本番用）

### authlibによるOAuth実装

```python
# app.py
from authlib.integrations.flask_client import OAuth

oauth = OAuth(app)
oauth.register(
    name="google",
    client_id=os.environ.get("GOOGLE_CLIENT_ID"),
    client_secret=os.environ.get("GOOGLE_CLIENT_SECRET"),
    server_metadata_url="https://accounts.google.com/.well-known/openid-configuration",
    client_kwargs={"scope": "openid email profile"},
)

@app.route("/auth/login")
def auth_login():
    state = os.urandom(16).hex()
    # Cloud Run でセッションCookieが消えることがあるためファイルにも保存
    _save_oauth_state_to_fs(state, GOOGLE_REDIRECT_URI)
    return oauth.google.authorize_redirect(GOOGLE_REDIRECT_URI, state=state)

@app.route("/auth/callback")
def auth_callback():
    state = request.args.get("state", "")
    _restore_oauth_state_from_fs(state)  # セッション復元
    try:
        token = oauth.google.authorize_access_token()
    except MismatchingStateError:
        return redirect(url_for("auth_login"))  # CSRF失敗時は再ログイン

    user = token.get("userinfo") or oauth.google.userinfo(token=token)
    session["user"] = {
        "sub":     user["sub"],      # Google UID（不変の識別子）
        "email":   user["email"],
        "name":    user.get("name", ""),
        "picture": user.get("picture", ""),
    }
    _migrate_root_data_to_user(user["sub"])  # 初回ログイン時データ移行
    return redirect("/")
```

### Cloud Runでセッションが消える問題の解決

Cloud Runはリクエストごとに異なるインスタンスで処理されることがあります。Google OAuthのstateパラメータがセッションから消えると `MismatchingStateError` が発生します。

**解決策: OAuthステートをファイルシステムにバックアップ**

```python
_OAUTH_STATE_DIR = Path(os.environ.get("YONDA_DATA_DIR", "data")) / ".oauth_states"

def _save_oauth_state_to_fs(state: str, redirect_uri: str) -> None:
    """ログイン開始時にstateをGCS(マウント済みディレクトリ)に保存"""
    _OAUTH_STATE_DIR.mkdir(parents=True, exist_ok=True)
    state_file = _OAUTH_STATE_DIR / f"oauth_state_{state}.json"
    state_file.write_text(json.dumps({
        "state": state,
        "redirect_uri": redirect_uri,
        "created_at": time.time(),
    }))

def _restore_oauth_state_from_fs(state: str) -> bool:
    """コールバック時にセッションにstateを復元"""
    state_key = f"_state_google_{state}"
    if state_key in session:
        return True  # セッションにある場合はそのまま
    state_file = _OAUTH_STATE_DIR / f"oauth_state_{state}.json"
    if not state_file.exists():
        return False
    data = json.loads(state_file.read_text())
    if time.time() - data["created_at"] > 600:  # 10分で期限切れ
        state_file.unlink(missing_ok=True)
        return False
    # authlibのセッション形式で復元
    session[state_key] = {
        "state": data["state"],
        "redirect_uri": data["redirect_uri"],
    }
    state_file.unlink(missing_ok=True)
    return True
```

### SECRET_KEYの一貫性確保

Flaskのセッションクッキーの署名にSECRET_KEYを使います。Cloud Runの複数インスタンスで異なるキーになるとセッションが無効化されるため、環境変数で統一します。

```bash
# Cloud Runの環境変数に設定
FLASK_SECRET_KEY=your-fixed-secret-key-do-not-change
```

```python
# app.py でのキー取得順序（フォールバック）
def _get_secret_key():
    # 1. 環境変数（本番推奨）
    if k := os.environ.get("FLASK_SECRET_KEY"):
        return k
    # 2. GOOGLE_CLIENT_SECRETから派生（一意性保証）
    if k := os.environ.get("GOOGLE_CLIENT_SECRET"):
        import hashlib
        return hashlib.sha256(k.encode()).hexdigest()
    # 3. ファイル（ローカル開発用）
    key_file = Path(os.environ.get("YONDA_DATA_DIR", "data")) / ".secret_key"
    if key_file.exists():
        return key_file.read_text().strip()
    # 4. 新規生成してファイルに保存
    new_key = os.urandom(32).hex()
    key_file.write_text(new_key)
    return new_key
```

### 認証ガード（before_request）

```python
_PUBLIC_PREFIXES = ("/auth/", "/static/", "/help")
_PUBLIC_EXACT = {"/", "/api/docs", "/api/messages", "/api/book-cover", "/api/book-info"}

@app.before_request
def _before_request_handler():
    library_service.set_user_data_dir(get_user_data_dir_for_session())

    if not _OAUTH_ENABLED:
        return  # OAuth未設定時は全公開

    path = request.path
    if any(path.startswith(p) for p in _PUBLIC_PREFIXES):
        return
    if path in _PUBLIC_EXACT:
        return

    if not get_current_user():

            return jsonify({"error": "認証が必要です"}), 401
        return redirect(url_for("auth_login"))
```

### 初回ログイン時のデータ移行

最初のユーザーが既存データを引き継ぐ仕組みです。2人目以降は空のデータから開始します。

```python
def _migrate_root_data_to_user(uid: str) -> None:
    uid_safe = re.sub(r"[^a-zA-Z0-9_\-]", "_", uid)
    sentinel = library_service.DATA_DIR / ".data_migrated"
    user_dir = library_service.DATA_DIR / "users" / uid_safe
    user_dir.mkdir(parents=True, exist_ok=True)

    migrate_files = [
        "audible_books.json", "library_books.json",
        "kindle_books.json", "paper_books.json",
        "book_insights.json", "private_books.json",
        "hidden_books.json", "amazon_list.json",
    ]

    # 既にユーザーデータがあればスキップ
    if any((user_dir / f).exists() for f in migrate_files):
        return

    # センチネルがあれば2人目以降 → 空のデータディレクトリで開始
    if sentinel.exists():
        logger.info("新規ユーザー(%s): 空から開始", uid_safe)
        return

    # 初回ユーザー: ルートのデータをユーザーディレクトリにコピー
    for fname in migrate_files:
        src = library_service.DATA_DIR / fname
        if src.exists():
            shutil.copy2(src, user_dir / fname)

    sentinel.touch()  # 以降のユーザーは空から開始するフラグ
    logger.info("初回ログイン移行完了: %s", uid_safe)
```

```python
# adapters/audible.py
import audible

class AudibleJPAdapter(LibraryAdapter):
    library_id = "audible_jp"
    library_name = "Audible Japan"
    needs_credentials = False  # auth_jp.json を使うため

    def login(self, session, credentials=None):
        # auth_jp.json（またはGCS Secret Manager）から認証
        auth_file = os.environ.get("YONDA_AUTH_FILE", "data/auth_jp.json")
        self._auth = audible.Authenticator.from_file(auth_file)
        self._client = audible.Client(auth=self._auth)
        return True
```

```python
            title_elem = item.select_one("h3 a")

            detail_url = self.library_url + title_elem["href"]
            # 図書館資料番号をURLから抽出

            data = {}
```

```python
def _fetch_summary_and_genre_from_google_books(
    title: str, author: str,
    isbn: Optional[str] = None,
    api_key: Optional[str] = None,
) -> tuple[Optional[str], Optional[str]]:

    def _search(params: dict):
        url = "https://www.googleapis.com/books/v1/volumes"
        if api_key:
            params["key"] = api_key
        resp = requests.get(url, params=params, timeout=8)
        items = resp.json().get("items", [])
        if not items:
            return None, None
        info = items[0].get("volumeInfo", {})
        summary = info.get("description", "")
        categories = info.get("categories", [])
        genre = categories[0] if categories else ""
        return summary or None, genre or None

    # ISBN検索が最も正確
    if isbn:
        result = _search({"q": f"isbn:{isbn}"})
        if result[0]:
            return result

    # タイトル+著者で検索
    return _search({"q": f"intitle:{title}+inauthor:{author}", "langRestrict": "ja"})
```

```python
def _fetch_summary_and_genre_from_open_library(
    title: str, author: str, isbn: Optional[str] = None
) -> tuple[Optional[str], Optional[str]]:

    # ISBNがある場合は直接取得
    if isbn:
        url = f"https://openlibrary.org/api/books?bibkeys=ISBN:{isbn}&format=json&jscmd=details"
        resp = requests.get(url, timeout=8)
        data = resp.json()
        if data:
            details = list(data.values())[0].get("details", {})
            return details.get("description", {}).get("value", ""), ""

    # タイトル検索
    url = "https://openlibrary.org/search.json"
    resp = requests.get(url, params={"title": title, "author": author, "limit": 3}, timeout=8)
    docs = resp.json().get("docs", [])
    if docs:
        key = docs[0].get("key", "")
        if key:
            work = requests.get(f"https://openlibrary.org{key}.json", timeout=8).json()
            desc = work.get("description", "")
            if isinstance(desc, dict):
                desc = desc.get("value", "")
            return desc or None, None
    return None, None
```

```python
def save_books(uid: str, source_id: str, books: list[dict], meta: dict) -> None:
    db = get_db()
    if not db:
        return  # Firestoreなしでも動作継続

    user_ref = db.collection("users").document(uid)
    books_col = user_ref.collection("books")
    now = datetime.now(timezone.utc).isoformat()

    batch = db.batch()
    count = 0

    for book in books:
        bid = make_book_id(book)
        clean = {k: v for k, v in book.items() if v is not None}
        clean["_updated_at"] = now
        batch.set(books_col.document(bid), clean)  # 上書き保存
        count += 1

        if count >= 499:  # 500件ごとにコミット
            batch.commit()
            batch = db.batch()
            count = 0

    if count > 0:
        batch.commit()

    # ソースメタデータを更新
    user_ref.collection("sources").document(source_id).set({**meta, "_updated_at": now})
```

```python
@app.route("/api/book-insights/generate", methods=["POST"])
def api_generate_book_insight():
    data = request.get_json()
    book = data.get("book", {})

    title = book.get("title", "")
    author = book.get("author", "")
    summary = book.get("full_summary") or book.get("summary", "")
    genre = book.get("genre", "")

    ai_config = library_service.load_ai_config()
    prompt = f"""
以下の本について、読者が「読んでよかった」と感じるポイントを3〜5個、
箇条書きで簡潔に教えてください。

タイトル: {title}
著者: {author}
ジャンル: {genre}
概要: {summary[:500]}

各ポイントは1〜2文で、具体的な学びや気づきを含めてください。
"""

    if ai_config.get("provider") == "openai":
        import openai
        client = openai.OpenAI(api_key=ai_config["api_key"])
        response = client.chat.completions.create(
            model="gpt-4o",
            messages=[{"role": "user", "content": prompt}],
            max_tokens=500,
        )
        insight_text = response.choices[0].message.content

    elif ai_config.get("provider") == "gemini":
        import google.generativeai as genai
        genai.configure(api_key=ai_config["api_key"])
        model = genai.GenerativeModel("gemini-1.5-flash")
        response = model.generate_content(prompt)
        insight_text = response.text

    # 保存
    insight = {"points": insight_text, "generated_at": datetime.now().isoformat()}
    library_service.save_book_insight(book, insight)

    return jsonify({"success": True, "insight": insight})
```

```javascript
const imageObserver = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
```

```python
        "has_key": bool(config.get("api_key")),
        # "api_key" は返さない！
    })
```

## 20. Cursor + Claudeを使った開発の進め方

### なぜCursor + Claudeか

このアプリは **Cursor（AIコードエディタ）とClaude（Anthropicの大規模言語モデル）** を使って短期間で開発しました。

```
開発期間の比較:
- 従来の開発: 2〜3ヶ月
- Cursor + Claude: 週末2日 + 細部の調整数日
```

### 効果的な使い方

#### 1. 全体設計をClaude に相談する

```
「PythonとFlaskで読書記録アプリを作りたい。
Audible・Kindle・図書館から本を自動取得して、
AIで書評を生成する機能が欲しい。
全体アーキテクチャを教えて」
```

Claudeはアーキテクチャ案・技術選定・実装方針を詳細に提案します。

#### 2. スモールスタートで始める

最初から完璧を目指さず、動くものを早く作ります。

```
Week 1 目標:
「Flask + Google OAuth + 書籍データのJSONファイル保存
 だけで動くものを作る」

Week 2 目標:
「Audible連携を追加する」

Week 3 目標:
「フロントエンドを整える」
```

#### 3. エラーはそのままClaudeに貼る

```
「以下のエラーが出ています。修正してください:

Traceback (most recent call last):
  File "app.py", line 45, in auth_callback
    token = oauth.google.authorize_access_token()
authlib.integrations.base_client.errors.MismatchingStateError:
CSRF Warning! State not equal in request and response.」
```

Claudeはエラーの原因と解決策を即座に提案します。このアプリ開発でも `MismatchingStateError` は実際に発生し、Claudeが提案したファイルシステムバックアップ方式で解決しました。

#### 4. コードの説明を求める

Claudeが生成したコードが理解できない場合は説明を求めます。

```
「このコードの threading.local() の部分が
なぜ必要か説明して」
```

#### 5. Cursorのマルチファイル編集を活用

Cursorは複数ファイルを同時に編集できます。「`app.py` と `library_service.py` の両方を修正して」という指示が1回で効きます。

#### 6. テストを先に書く

```
「/api/books エンドポイントのユニットテストを
pytest で書いて。ログイン済み・未ログインの
両方のケースをカバーして」
```

### 実際に詰まったポイントと解決策

| 問題 | 原因 | 解決策 |
|------|------|--------|
| Cloud RunでMismatchingStateError | セッションCookieがインスタンス間で共有されない | OAuthステートをGCSにファイルで保存 |
| Kindleデータが取得できない | AmazonのOTP認証 | セッションストア + OTPフロー実装 |
| 2,000冊表示でブラウザが固まる | DOM操作が重い | ページネーション + 遅延読み込み |
| Audible認証ファイルの期限切れ | auth_jp.jsonの有効期限 | Secret Manager + 定期更新フロー |
| JSONファイルの破損 | 複数インスタンスの同時書き込み | Firestoreへの移行 + Extra dataエラーの安全な処理 |
