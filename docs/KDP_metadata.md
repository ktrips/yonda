## 内容説明（商品説明欄・約 1,100 字）

```
作りたいアプリはあるのに、平日は時間がない。
本書は、AIエディタ（Cursor + Claude）を使って「週末だけ」で実用アプリを作り、
さらにそれを収益につなげるところまでを、実在のアプリでまるごと見せる実践ガイドです。

題材は、著者が実際に週末開発で作り、いま本番稼働している読書記録アプリ「Yonda」。
Audible・Kindle・公共図書館・紙の本の記録を毎朝自動で同期し、
AIが書評と次に読む本まで提案する——読書が、あなたの財産になるアプリです。

■ 本書で学べること
・Cursor + Claude（Fable 5 世代）への「頼み方」——仕様の渡し方、分割の仕方、詰まったときの聞き方
・Audible 非公式 API・Kindle・図書館スクレイピングという「面倒な連携」の突破法
・Google ログインとマルチユーザー設計、Cloud Scheduler での全ユーザー自動同期
・Google Cloud Run へのデプロイと、月数百円で運用するコスト設計
・テスト・監視・セキュリティ・トラブルシューティングの実務

■ 収益化編（28〜30章）
「作って終わり」にしないための3章。ティール・ベゾス・アルトマン・マスク・ジョブズの
5つの視点で戦略を設計し、アフィリエイト・KDP出版・サブスク（Stripe）を実装し、
48時間実験と資産化で週末開発を複利のループに変えます。
本書自体がそのループの実例です——アプリの開発記がこの本になり、
この本が読者をアプリに連れてきます。

■ 対象読者
・Python の基礎（関数・クラス）がわかる方
・Webアプリ開発は初めて、または独学の方
・生成AIを開発にもっと活用したい方
・趣味の開発を本番公開し、小さくても収益を生む形にしたい方

■ 構成
第I部「Webアプリを作る — 開発手法」（1〜27章）＋
第II部「Webアプリを収益化する — マーケティング手法」（28〜30章）＋付録。
冒頭の「Yondaで学ぶ」対応表で、作る機能と身につくスキルが一覧できます。
コードは「写経」より「理解」重視。各所に Claude への質問例を載せているので、
詰まったらそのままAIに聞きながら進められます。

完成したアプリは yonda.ktrips.net で実際に触れます。
まず動くものを見てから、週末のあなたの手で同じものを作ってください。
```
